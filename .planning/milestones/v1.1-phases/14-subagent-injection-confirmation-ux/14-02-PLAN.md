**Step A: Add shared injection instruction block.**

Add a new subsection (after the Initialize section, before "Parse and Normalize Arguments") that describes how to read wizard-state.json and build capability blocks. This instruction runs ONCE per workflow invocation:

```markdown
## 1.5. Load Toolkit Capabilities

Read `.claude/wizard-state.json` if it exists. Extract `toolkit.by_stage` from the JSON.

- If the file is missing, `toolkit` is empty (`{}`), or `by_stage` is absent: set TOOLKIT_AVAILABLE = false. No capability blocks will be built for any spawn in this workflow.
- If by_stage is present and non-empty: set TOOLKIT_AVAILABLE = true. Store the by_stage data for use when building capability blocks below.

**Build Capability Block (reused by spawn points below):**

When a spawn point below says "build capability block for stage {STAGE}":

1. If TOOLKIT_AVAILABLE is false: skip -- do not append any block.
2. Read by_stage.{STAGE} (array of tool names).
3. If the array is empty: skip.
4. For each tool name in the array:
   - If name matches a known MCP tool (context7, deepwiki, or any name recognized as an MCP server):
     format as: `- {name} (configured -- availability may vary) -- MCP server for {name}`
   - Otherwise: format as: `- {name} -- {tool name as identifier}`
5. Wrap with the stage-appropriate preamble:

<capabilities>
{preamble}
{pointer lines}
</capabilities>

Stage preambles:
  - research: "Query these before investigating unknowns:"
  - planning: "Reference these during planning if relevant:"
  - execution: "Use these during implementation if relevant:"
  - review: "Use these for validation and checking:"

6. Append this block to the Task() prompt, after `<files_to_read>` and before `<output>` or `<success_criteria>`.
7. Never read toolkit-registry.json -- only wizard-state.json is used (PERF-03).
```
