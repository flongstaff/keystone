Add a conditional toolkit counts line to the status box section of wizard-detect.sh. Two changes:

**1. Extract counts from TOOLKIT_JSON for status box display (add after the TOOLKIT_JSON capture, before the status box section):**

Use a Python3 one-liner (consistent with established pattern in wizard-detect.sh — see IS_RESET detection at line 263 and infra safety injection at line 163) to extract individual counts:

```bash
TOOLKIT_LINE=""
if [ "$TOOLKIT_JSON" != '{}' ]; then
    TOOLKIT_LINE=$(python3 -c "
import json, sys
try:
    d = json.loads('$TOOLKIT_JSON')
    c = d.get('counts', {})
    parts = []
    if c.get('agents', 0) > 0: parts.append(f\"{c['agents']} agents\")
    if c.get('skills', 0) > 0: parts.append(f\"{c['skills']} skills\")
    if c.get('hooks', 0) > 0: parts.append(f\"{c['hooks']} hooks\")
    if c.get('mcp', 0) > 0: parts.append(f\"{c['mcp']} MCP\")
    print(', '.join(parts))
except Exception:
    pass
" 2>/dev/null)
fi
```

CRITICAL: The TOOLKIT_JSON variable may contain characters that break bash string interpolation inside the python3 -c "..." string. Instead of embedding $TOOLKIT_JSON directly in the Python code, pipe it via stdin or use an environment variable:

```bash
TOOLKIT_LINE=""
if [ "$TOOLKIT_JSON" != '{}' ]; then
    TOOLKIT_LINE=$(echo "$TOOLKIT_JSON" | python3 -c "
import json, sys
try:
    d = json.load(sys.stdin)
    c = d.get('counts', {})
    parts = []
    if c.get('agents', 0) > 0: parts.append(f\"{c['agents']} agents\")
    if c.get('skills', 0) > 0: parts.append(f\"{c['skills']} skills\")
    if c.get('hooks', 0) > 0: parts.append(f\"{c['hooks']} hooks\")
    if c.get('mcp', 0) > 0: parts.append(f\"{c['mcp']} MCP\")
    print(', '.join(parts))
except Exception:
    pass
" 2>/dev/null)
fi
```
